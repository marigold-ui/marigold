The `<DateRangePicker>` component lets users enter or select a date range — a start and an end date — through a single field.

Like `<DatePicker>`, it pairs a date input with a calendar popover. Clicking the calendar button opens a `<RangeCalendar>` where the user can select the range, while the start and end can also be typed or pasted directly into the field.

## Anatomy

The Label defines the purpose of the field. Two date inputs, separated by a dash, hold the start and end of the range. Tapping the Calendar Button opens a range calendar with a Header for selecting the month and year, and Next/Previous Buttons for month navigation.

## Usage

`<DateRangePicker>` is used when users need to select a span of time accurately — reporting periods, event windows, booking ranges, or availability filters. It standardizes input across locales and prevents invalid selections through built-in validation. Avoid it for selecting a single date (use `<DatePicker>`) or for casual input where a plain text field would suffice.

### Basic Usage (uncontrolled)

This example shows a regular `<DateRangePicker>` without any special props.

<ComponentDemo name="date-range-picker-basic" />

### Min/Max Values

The `minValue` and `maxValue` props are used to perform built-in validation. This prevents the user from selecting dates outside the valid range in the calendar.

<ComponentDemo name="date-range-picker-min-max" />

### Controlled

The `value` and `onChange` props can be used to control the `DateRangePicker`. The value is a `RangeValue<DateValue>` — an object with a `start` and `end` date.

<ComponentDemo name="date-range-picker-controlled" />

### Multiple Months

Use `visibleDuration` to display up to three months at once in the popover, which makes selecting longer ranges easier. On small screens the calendar stacks into a single month inside a tray.

<ComponentDemo name="date-range-picker-multi-month" />

### Error

Set `error` and provide an `errorMessage` to mark the field as invalid. The error message is shown instead of the description.

<ComponentDemo name="date-range-picker-error" />

### Paste Support

The `<DateRangePicker>` supports pasting date values from the clipboard into either the start or end input. Paste into an input and the component parses and validates the content, setting that part of the range.

**Supported date formats:**

- ISO format: `2023-12-25`
- European format: `25.12.2023` or `25/12/2023`
- US format: `12/25/2023` or `12-25-2023`

Invalid dates or unrecognized formats are ignored, ensuring data integrity.
