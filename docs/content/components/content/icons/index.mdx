This page explains how to install and use Marigold icons. For the principles
behind icon design and the catalog of available icons, see
[Iconography](/foundations/iconography/).

## Installation

Marigold icons are distributed as `@marigold/icons`. Add it to your project:

```bash
# with npm
npm install @marigold/icons --save

# with pnpm
pnpm add @marigold/icons
```

The package re-exports every icon from
[`lucide-react`](https://lucide.dev/) and adds a small set of brand-specific
icons on top.

## Importing icons

Import icons by name from `@marigold/icons`:

```tsx
import { DesignTicket, Search } from '@marigold/icons';

export default function MyComponent() {
  return (
    <>
      <Search />
      <DesignTicket />
    </>
  );
}
```

### Brand icons

These icons are Marigold-specific and only exist in `@marigold/icons`:
`DesignTicket`, `GiftCard`, `PDF`, `Resale`, `Scanner`, `Stadium`,
`TicketInsurance`, `Turnstile`, `Facebook`, `Google`, `Instagram`, `Twitter`.

Everything else comes from Lucide — see [lucide.dev](https://lucide.dev/) for
the full catalog.

## Sizing and color

Icons accept Lucide's standard props: `size` (number of pixels),
`color` (CSS color or token), `strokeWidth`, and any other SVG attribute.

## Accessibility

Icons paired with a visible label should be marked `aria-hidden` so screen
readers don't repeat the label. Icon-only interactive elements need an
`aria-label` describing the action:

```tsx
<button aria-label="Delete">
  <Trash2 aria-hidden />
</button>
```

## Usage

Icons are used to enhance the user interface by providing visual cues and
improving the overall user experience. They can be used in buttons, links, or
as standalone elements to represent actions, statuses, or concepts. Icons
should be used sparingly and only when they add value to the user interface.

## Migrating from the legacy icon set

<Callout title="Migrating from older versions of @marigold/icons?">
  The package name and import path are unchanged: `@marigold/icons`. Most icons
  now come from Lucide, which means some legacy names map to a Lucide equivalent
  — see the [Iconography](/foundations/iconography/) catalog for the new names.
  Brand icons (listed above) are kept as-is. The `size` prop now accepts a
  number of pixels rather than a CSS unit; if you previously passed a string
  like `"24px"`, switch to the number `24` or use class-based sizing.
</Callout>
