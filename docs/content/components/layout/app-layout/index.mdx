The `<AppLayout>` creates an L-shape application layout using CSS Grid. It places a sidebar on the left spanning the full height, with a header and scrollable main content area stacked on the right. The layout fills the full viewport height and prevents the page from scrolling. Only the main content area scrolls.

## Usage

Use `<AppLayout>` when your application needs a persistent sidebar alongside a top navigation bar, the classic "L-shape" layout found in admin panels, dashboards, and enterprise tools.

### Basic structure

The three slot components can be placed in any order as children of `<AppLayout>`. The CSS Grid template ensures they render in the correct positions.

```tsx
<AppLayout>
  <AppLayout.Sidebar>{/* Sidebar content */}</AppLayout.Sidebar>
  <AppLayout.Header>{/* Top navigation */}</AppLayout.Header>
  <AppLayout.Main>{/* Scrollable page content */}</AppLayout.Main>
</AppLayout>
```

### Integration with Sidebar and TopNavigation

`<AppLayout>` is designed to work with the [`<Sidebar>`](/components/navigation/sidebar) and [`<TopNavigation>`](/components/navigation/topnavigation) components. For a complete guide on composing these components together, including content placement, provider nesting, and responsive behavior, see the [App Shell pattern](/patterns/app-shell).

### Scrolling

The sidebar and header always stay visible, no matter how far users scroll. Only the content inside `<AppLayout.Main>` is scrollable. Because the grid keeps everything else in place, components inside the header or sidebar don't need their own sticky behavior.

### Responsive behavior

`<AppLayout>` itself does not change its grid structure at different breakpoints. It always renders the three-area L-shape. Responsive behavior comes from the components placed inside of the slots.

### Accessibility

- **Landmarks:** `<AppLayout.Main>` renders as `<main>`, providing a standard landmark region for screen readers. The sidebar landmark is provided by the `<Sidebar>` component placed inside `<AppLayout.Sidebar>`
- **Scrollable region:** The main area is scrollable and can be navigated with keyboard (arrow keys, Page Up/Down, Home/End)
- **Stacking order:** The header uses `z-index: 1` to ensure it stays above main content during scroll

<Callout title="Note">
  `<AppLayout>` is a structural layout component. Accessibility features like keyboard navigation, focus management, and ARIA attributes are provided by the components placed inside it. See [Sidebar](/components/navigation/sidebar) and [TopNavigation](/components/navigation/topnavigation) for their accessibility details.
</Callout>
