A `<Card>` component is a versatile UI element used to display content in a concise, visually organized format.
It's typically presented as a rectangular box with various content elements such as images, text, buttons, and icons.
Cards are ideal for grouping related information and making it easily digestible, often used to display previews of articles, products, user profiles, and other content types.

## Anatomy

The `<Card>` component is a compound component made up of several sub-components that map to distinct visual areas of the card:

- **Card.Preview**: Optional full-bleed area at the top, typically used for images or media.
- **Card.Header**: Area for the card title and supporting labels or badges.
- **Card.Body**: Main content area for descriptive text or form elements.
- **Card.Footer**: Bottom area for actions such as buttons or metadata.

The Card component supports variants for general content grouping as well as access-level marking.

| Variant   | Description                                                      | When to use                                                                               |
| --------- | ---------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| `default` | Standard card with a raised shadow. This is the default variant. | General-purpose content grouping like articles, profiles, or summaries.                   |
| `admin`   | Card with admin-specific border and background tint.             | Grouping content visible only to admin users. [Learn more.](/patterns/admin-master-mark)  |
| `master`  | Card with master-specific border and background tint.            | Grouping content visible only to master users. [Learn more.](/patterns/admin-master-mark) |

## Usage

The `<Card>` component enhances the visual hierarchy and organization of content, making it easier for users to engage with various pieces of information.

Cards use a compound component pattern, you compose the card layout using the provided sub-components. Not all slots need to be used: combine only the ones relevant to your content.

### Card.Preview

Use `Card.Preview` to display an image or thumbnail that stretches edge-to-edge across the top of the card, without any padding. Typical use cases are venue photos, product images, or event covers.

<ComponentDemo name="card-elevation" />

### Card.Header

Use `Card.Header` for the card's title and supporting meta information like badges or labels.

```tsx
<Card>
  <Card.Header>
    <Headline level={3}>Event Title</Headline>
    <Badge variant="info">New</Badge>
  </Card.Header>
  <Card.Body>Details about the event.</Card.Body>
</Card>
```

### Card.Body

Use `Card.Body` for the main content area of the card, descriptive text, form elements, or any primary content.

<GuidelineTiles>
  <Do>
    <DoDescription>
      Use `Card.Preview` for images so they appear full-bleed at the top of the

    </DoDescription>
  </Do>
  <Dont>
    <DontDescription>
      Don't nest images inside `Card.Body` when they're meant to be the card's

  </Dont>
</GuidelineTiles>

### Card.Footer

Use `Card.Footer` for actions or metadata displayed at the bottom of the card, such as buttons or secondary text.

```tsx
<Card>
  <Card.Body>Card content</Card.Body>
  <Card.Footer>
    <Button>Primary Action</Button>
  </Card.Footer>
</Card>
```
