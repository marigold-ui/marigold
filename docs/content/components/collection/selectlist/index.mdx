Use `SelectList` when each option needs a description line to help the user decide. For short, self-explanatory choices, prefer `RadioGroup` (single) or `CheckboxGroup` (multiple).

## Anatomy

A `SelectList` consists of a container with a stack of `SelectList.Option` rows. Each row has a selection indicator (radio or checkbox), a label stacked over an optional description, and an optional trailing action.

## Usage

Build each row with `<Text slot="label">` and `<Text slot="description">` — the same slot convention used by `Select`, `ComboBox`, and `Autocomplete`. The theme handles placement, typography, and wiring the description to the row's `aria-describedby`.

When children are not a plain string (for example, when you wrap the label in `<Text slot="label">` or add an action), pass `textValue` so screen readers announce the correct option name.

When `selectionMode="single"`, rows render a radio circle. When `selectionMode="multiple"`, rows render a checkbox.

### Bordered variant

Use `variant="bordered"` when each option represents a commit-level choice (plans, shipping methods, payment modes). Each row becomes its own outlined container, and the selected row's border strengthens instead of filling the row.

<ComponentDemo name="selectlist-bordered" />

### Multiple selection with an action

Set `selectionMode="multiple"` to let users pick more than one row. Rows then render a checkbox indicator instead of a radio circle.

Drop an `<ActionMenu>` or `<IconButton>` directly inside a `SelectList.Option` to place a trailing action. The SelectList styles and positions it automatically — no wrapper needed. Use at most **one action per option**.

<ComponentDemo name="selectlist-multiple-selection" />

### Disabled behavior

The `<SelectList>` lets you disable certain options, which is helpful if you don't want users to select them. You can use the `disabledBehavior` prop to control what happens with those disabled options. In the example below, we use `disabledBehavior="selection"`. This means the disabled options can't be selected and won't show a selection indicator — but you can still use their actions (like buttons or menus inside them).

## Alternative components

- **RadioGroup / CheckboxGroup** — when options are short, text-only, and always visible. Use a radio group for a small, fixed set of mutually exclusive choices, and a checkbox group for independent multi-select boolean options.
- **ListBox** — when the user is picking an option from a compact, dropdown-like list inside a `Select`, `ComboBox`, or `Autocomplete`. `ListBox` rows are single-line and dense; they don't have a standardised label / description skeleton.
- **Table** — when each row has several columns of structured data and headers matter. Use `Table` for selectable data grids.
