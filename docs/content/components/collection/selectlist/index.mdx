The `SelectList` lets users pick one or more items from a list of rich, two-line options. Unlike `ListBox` — which is optimised for dense dropdown menus — each `SelectList.Item` is a standardised row with a selection indicator, an optional leading image, a label, a description, and an optional trailing action.

## Anatomy

A `SelectList` consists of a container with a stack of `SelectList.Item` rows. Each row follows a fixed skeleton: a selection indicator (radio or checkbox) at the start, an optional image, a label stacked over a description, and an optional trailing action.

## Usage

Build each row with `<Text slot="label">` and `<Text slot="description">` — the same slot convention used by `Select`, `ComboBox`, and `Autocomplete`. The theme handles placement, typography, and wiring the description to the row's `aria-describedby`.

When `selectionMode="single"`, rows render a radio circle. When `selectionMode="multiple"`, rows render a checkbox.

### Bordered variant

Use `variant="bordered"` when each option represents a commit-level choice (plans, shipping methods, payment modes). Each row becomes its own outlined container, and the selected row's border strengthens instead of filling the row.

<ComponentDemo name="selectlist-bordered" />

### With a leading image

Drop any `<img>` (or icon component) as a child to render it as a leading visual. Use `alt=""` when the image is decorative — the label already names the row.

<ComponentDemo name="selectlist-image" />

### Multiple selection

Set `selectionMode="multiple"` to let users pick more than one row. Rows then render a checkbox indicator instead of a radio circle.

You can also provide interactive actions for each item by using `<SelectList.Action>`, such as buttons or menus, to perform specific actions on the row.

<ComponentDemo name="selectlist-multiple-selection" />

### Disabled behavior

The `<SelectList>` lets you disable certain items, which is helpful if you don't want users to select them. You can use the `disabledBehavior` prop to control what happens with those disabled items. In the example below, we use `disabledBehavior="selection"`. This means the disabled items can't be selected and won't show a selection indicator — but you can still use their actions (like buttons or links inside them).

## Alternative components

- **RadioGroup / CheckboxGroup** — when options are short, text-only, and always visible. Use a radio group for a small, fixed set of mutually exclusive choices, and a checkbox group for independent multi-select boolean options.
- **ListBox** — when the user is picking an option from a compact, dropdown-like list inside a `Select`, `ComboBox`, or `Autocomplete`. `ListBox` rows are single-line and dense; they don't have a standardised label / description / image skeleton.
- **Table** — when each row has several columns of structured data and headers matter. Use `Table` for selectable data grids.
