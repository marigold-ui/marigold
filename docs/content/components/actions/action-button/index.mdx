The `<ActionButton>` component is a low-emphasis button intended to live inside another surface like a toolbar, a list row, the header of a `<Panel>` or `<EmptyState>`, or the chrome of a `<Dialog>`. It defaults to the `ghost` variant so the button recedes into its surroundings rather than competing with the page's primary action.

For the primary action of a page or form, reach for [`<Button>`](/components/actions/button). To cluster related actions in a toolbar, use [`<ActionGroup>`](/components/actions/action-group). For the bulk-action chrome inside `<ActionBar>`, use [`<ActionBar.Button>`](/components/actions/actionbar). For more actions than fit comfortably in a row, use [`<ActionMenu>`](/components/overlay/action-menu).

ActionButton reuses [`<Button>`](/components/actions/button)'s theme classes, so the two stay visually aligned. The variant set is narrower because ActionButton is meant to recede. For the bold variants (`primary`, `destructive`) reach for `<Button>` instead.

| Variant             | Description                                                    | When to use                                                                                                                                                 |
| ------------------- | -------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ghost`             | Transparent, no border or background. **The default variant.** | Toolbar actions, list-row actions, container headers, or anywhere the button should recede into the surrounding UI.                                         |
| `secondary`         | Neutral button with a subtle border.                           | When the action needs slightly more visual weight than `ghost` without competing with a primary button on the same surface.                                 |
| `destructive-ghost` | Red text with no background.                                   | Low-emphasis destructive actions, like a "Remove" button on a list item. See [the cascade rules](/components/actions/action-group#cascade-rules) for usage. |
| `link`              | Styled as an inline text link.                                 | Actions that should blend into surrounding text while remaining interactive.                                                                                |

## Usage

ActionButton is the action that belongs to a surface, not to the page. It lives inside a row, a toolbar, or a container header and recedes into that surface.

<GuidelineTiles>
  <Do>
    <DoDescription>
      Pair icon-only ActionButtons with a `<Tooltip>` and an `aria-label` so the
      action is discoverable for sighted and assistive users alike.
    </DoDescription>
  </Do>
  <Dont>
    <DontDescription>
      Don't use `<ActionButton>` for the primary action of a page or form. Use
      [`<Button>`](/components/actions/button) instead.
    </DontDescription>
  </Dont>
</GuidelineTiles>

For a cluster of related actions in a row or toolbar, wrap them in an [`<ActionGroup>`](/components/actions/action-group). The group exposes a `role="toolbar"`, enables arrow-key navigation, and cascades shared `size`, `variant`, and `disabled` defaults to its children.

## Alternative components

- [ActionGroup](/components/actions/action-group): Use to cluster related actions in a toolbar.
- [Button](/components/actions/button): Use for the primary action of a page or form.
- [ActionBar](/components/actions/actionbar): Use for the bar around bulk-selection actions. The inner buttons are written as `<ActionBar.Button>`.
- [ToggleButton](/components/actions/toggle-button): Use when the button has an on/off state.
- [ActionMenu](/components/overlay/action-menu): Use the kebab-menu pattern when there are too many actions to fit in a Group.
