The `<ActionButton>` component is a low-emphasis button intended to live inside another surface like a toolbar, a list row, the header of a `<Panel>` or `<EmptyState>`, or the chrome of a `<Dialog>`. It defaults to the `ghost` variant so the button recedes into its surroundings rather than competing with the page's primary action.

For the primary action of a page or form, reach for [`<Button>`](/components/actions/button). To cluster related actions in a toolbar, use [`<ActionGroup>`](/components/actions/action-group). For the bulk-action chrome inside `<ActionBar>`, use [`<ActionBar.Button>`](/components/actions/actionbar). For more actions than fit comfortably in a row, use [`<ActionMenu>`](/components/overlay/action-menu).

## Anatomy

An ActionButton is a single pressable area that fires an action. It can carry a label, an optional leading icon, or render as an icon-only square for compact spaces like row toolbars.

<ActionButtonAnatomy />

- **Button:** The pressable area that triggers the action.
- **Label:** The text that names the action.
- **Icon (optional):** A leading icon paired with the label, or the sole visual in the icon-only form.
- **Icon-only:** A condensed square form for compact spaces. Always pair it with a `<Tooltip>` and an `aria-label` so the action stays discoverable for sighted and assistive users alike.

## Appearance

ActionButton reuses [`<Button>`](/components/actions/button)'s theme classes, so the two stay visually aligned. The variant set is narrower because ActionButton is meant to recede. For the bold variants (`primary`, `destructive`) reach for `<Button>` instead.

| Variant             | Description                                                    | When to use                                                                                                                 |
| ------------------- | -------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `ghost`             | Transparent, no border or background. **The default variant.** | Toolbar actions, list-row actions, container headers, or anywhere the button should recede into the surrounding UI.         |
| `secondary`         | Neutral button with a subtle border.                           | When the action needs slightly more visual weight than `ghost` without competing with a primary button on the same surface. |
| `destructive-ghost` | Red text with no background.                                   | Low-emphasis destructive actions, like a "Remove" button on a list item.                                                    |
| `link`              | Styled as an inline text link.                                 | Actions that should blend into surrounding text while remaining interactive.                                                |

## Usage

ActionButton is the action that belongs to a surface, not to the page. It lives inside a row, a toolbar, or a container header and recedes into that surface.

<GuidelineTiles>
  <Do>

      Pair icon-only ActionButtons with a `<Tooltip>` and an `aria-label` so the
      action is discoverable for sighted and assistive users alike.
    </DoDescription>
  </Do>
  <Dont>
    <DontDescription>
      Don't use `<ActionButton>` for the primary action of a page or form. Use
      [`<Button>`](/components/actions/button) instead.
    </DontDescription>
  </Dont>
</GuidelineTiles>

<Callout title="More than one action?">
  Wrap a cluster of related actions in an
  [`<ActionGroup>`](/components/actions/action-group). It keeps the buttons
  visually consistent and signals to users that the actions belong together
  as a single set.
</Callout>

### Button vs. ActionButton

`<ActionButton>` and [`<Button>`](/components/actions/button) look superficially similar, but they play different roles. `<Button>` is the **page-level commitment**, like submitting a form, saving a draft, or going to checkout. `<ActionButton>` is the **chrome of a surface**, like bookmarking a panel, removing a row, or editing a list item. The default variant reinforces the role. `<Button>` defaults to `secondary` so it reads as a real ask, while `<ActionButton>` defaults to `ghost` so it recedes into its container.

Pick the component by the role of the action, not by the variant you have in mind:

| Situation                                                     | Component                                                                                                                                                                                                                                 |
| ------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Submit a form, confirm a dialog, finish a wizard step         | [`<Button>`](/components/actions/button)                                                                                                                                                                                                  |
| The single most important call-to-action on a page or section | [`<Button variant="primary">`](/components/actions/button)                                                                                                                                                                                |
| Per-row action in a list, table, or `<SelectList>` option     | `<ActionButton>` inside an [`<ActionGroup>`](/components/actions/action-group). For actions that navigate (e.g. an "Edit" link to a detail page), use [`<LinkButton>`](/components/actions/link-button), which picks up the same cascade. |
| Chrome on a `<Panel>`, `<Dialog>`, or `<Drawer>` header       | `<ActionButton>` inside an [`<ActionGroup>`](/components/actions/action-group)                                                                                                                                                            |
| Bulk-selection bar around a table                             | [`<ActionBar.Button>`](/components/actions/actionbar)                                                                                                                                                                                     |
| Overflow when there are more actions than fit comfortably     | [`<ActionMenu>`](/components/overlay/action-menu)                                                                                                                                                                                         |
| Navigation to another page                                    | [`<LinkButton>`](/components/actions/link-button) or [`<Link>`](/components/actions/link)                                                                                                                                                 |

The example below shows both inhabiting the same surface. The header carries one `<Button>` for the page-level commitment ("Invite member"), and each row carries an `<ActionGroup>` of `<ActionButton>`s that recede into the row until the user scans for them.

### In context

ActionButton is at its best when it lives inside another component. Drop it into a `<SelectList.Option>`, a `<Panel.Header>`, an `<ActionBar>`, or anywhere a container publishes its own `ActionButtonContext`, and the container takes care of size, weight, and placement. The call site stays free of repetitive `variant` and `size` props.

## Alternative components

- [ActionGroup](/components/actions/action-group): Use to cluster related actions in a toolbar.
- [Button](/components/actions/button): Use for the primary action of a page or form.
- [LinkButton](/components/actions/link-button): The navigating sibling. Drop one into the same container or `<ActionGroup>` to render an action that takes the user to another page.
- [ActionBar](/components/actions/actionbar): Use for the bar around bulk-selection actions. The inner buttons are written as `<ActionBar.Button>`.
- [ToggleButton](/components/actions/toggle-button): Use when the button has an on/off state.
- [ActionMenu](/components/overlay/action-menu): Use the kebab-menu pattern when there are too many actions to fit in a Group.
