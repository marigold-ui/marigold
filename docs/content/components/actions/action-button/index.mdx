The `<ActionButton>` component is a low-emphasis button used in toolbars, list rows, and the headers and footers of containers like `<Panel>`, `<EmptyState>`, and `<Dialog>`. It defaults to the `ghost` variant so the button recedes into the surrounding UI rather than competing with the content.

Group related actions with `<ActionButton.Group>`. The group renders an accessible toolbar with arrow-key navigation, and it can publish defaults like size and variant that cascade to every button inside.

For the primary action of a page or form, reach for [`<Button>`](/components/actions/button) instead. For the bulk-action chrome inside `<ActionBar>`, use [`<ActionBar.Button>`](/components/actions/actionbar).

## Appearance

<AppearanceDemo component={'ActionButton'} />

<AppearanceTable component={'ActionButton'} />

ActionButton shares its variants and sizes with [`<Button>`](/components/actions/button), so the two stay visually consistent. The default variant is `ghost`.

| Variant             | Description                                                    | When to use                                                                                                                            |
| ------------------- | -------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| `ghost`             | Transparent, no border or background. **The default variant.** | Toolbar actions, list-row actions, container headers, anywhere the button should recede into the surrounding UI.                       |
| `secondary`         | Neutral button with a subtle border.                           | When the action needs slightly more visual weight than ghost without competing with a primary button.                                  |
| `primary`           | High-contrast filled button.                                   | Rare for ActionButton. Reserve for cases where the action is the most important on a low-chrome surface.                               |
| `destructive`       | Red filled button signalling a dangerous action.               | A destructive action inside a toolbar or row. See [the precedence example below](#actionbuttongroup) for mixing it into a ghost group. |
| `destructive-ghost` | Red text with no background.                                   | Low-emphasis destructive actions, like a "Remove" button on individual list items.                                                     |
| `link`              | Styled as an inline text link.                                 | Actions that should blend into surrounding text while remaining interactive.                                                           |

## Usage

ActionButton sits between `<Button>` (the primary action on a page or form) and `<ActionBar.Button>` (the buttons inside a bulk-action bar). Reach for it whenever an action belongs to a specific surface, like a Panel header, an EmptyState, a row in a list, or an item in a toolbar.

#### Cascade rules

The group publishes `size`, `variant`, and `disabled` to its children. Each prop has its own precedence rule, picked to match the most common case:

| Prop       | Winner | Why                                                                                                                         |
| ---------- | ------ | --------------------------------------------------------------------------------------------------------------------------- |
| `size`     | Group  | A toolbar should look uniform. Every button stays the same size unless you opt out by setting `size` directly on the child. |
| `variant`  | Local  | Lets you mix a `destructive` button into a `ghost` group without forcing every other button to opt out.                     |
| `disabled` | Local  | The group can disable the whole cluster, but a child can opt back in with `disabled={false}`.                               |

## Use inside a container

A container can publish shared defaults for the ActionButtons inside. A Panel header, for example, might say "every ActionButton in here is `ghost` and `small`" so each individual button stays free of repetitive props.

## Alternative components

- [Button](/components/actions/button): Use for primary actions on a page or form.
- [ActionBar](/components/actions/actionbar): Use for the bar around bulk-selection actions. The inner buttons are written as `<ActionBar.Button>`.
- [ToggleButton](/components/actions/toggle-button): Use when the button has an on/off state.
- [ActionMenu](/components/overlay/action-menu): Use the kebab-menu pattern when there are too many actions to fit in a Group.
