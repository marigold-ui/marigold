# Button

## Description

With the Button component you can render a `button` element.
The style variant and size of the button can be added with the variant prop and is composed like `variant.size`.

## Properties

| Property              | Type                                    | Default   |
| :-------------------- | :-------------------------------------- | :-------- |
| `variant` (optional)  | `primary`, `secondary`, `ghost`, `text` | `primary` |
| `size` (optional)     | `small`, `large`                        | `large`   |
| `disabled` (optional) | boolean                                 |           |
| `space` (optional)    | `ResponsiveStyleValue<string>`          | `none`    |

## Import

```tsx onlyCode
import { Button } from '@marigold/components';
```

## Usage

### Primary Button

The primary button function is to achieve the most common or most important goal of the user, it has more visual weight than the secondary options. There should not be more than one primary button in a screen at the time.

<br />

#### Default

```tsx
<Button>Primary</Button>
```

#### Disabled

```tsx
<Button disabled>Primary</Button>
```

### Secondary Button

Secondary buttons are an alternative to the user to the Primary option. They are used for actions like canceling, skipping, resetting, declining an offer etc.

<br />

#### Default

```tsx
<Button variant="secondary">Secondary</Button>
```

#### Disabled

```tsx
<Button variant="secondary" disabled>
  Secondary
</Button>
```

### Ghost Button

They are used for miscellaneous actions, they are important but is not the ones where it is desired that the user focus at the moment.

<br />

#### Default

```tsx
<Button variant="ghost">Ghost</Button>
```

#### Disabled

```tsx
<Button variant="ghost" disabled>
  Ghost
</Button>
```

### Suplementary Text Button

They can be used in a similar way of a secondary button, but they are meant to be less relevant to the user.

<br />

#### Default

```tsx
<Button variant="text">Text Only</Button>
```

#### Disabled

```tsx
<Button variant="text" disabled>
  Text Only
</Button>
```

### Button with icon

Icons can be added in Order to make clearer what a button does. Buttons should not be made only with icons, because every user may understand them on a different way.

```tsx
<Button space="xxsmall">
  <Ticket /> Icon Primary
</Button>
```

```tsx
<Button space="xxsmall" disabled>
  <Ticket /> Icon Primary
</Button>
```

### Small Buttons

If there are many actions in one page, is it possible to use small buttons instead of big ones. As well if the page is required to be open on mobile devices.

```tsx
<Button size="small">Primary</Button>
```

```tsx
<Button size="small" disabled>
  Primary
</Button>
```

```tsx
<Button variant="secondary" size="small">
  Secondary
</Button>
```

```tsx
<Button variant="secondary" size="small" disabled>
  Secondary
</Button>
```

```tsx
<Button variant="ghost" size="small">
  Ghost
</Button>
```

```tsx
<Button variant="ghost" size="small" disabled>
  Ghost
</Button>
```

```tsx
<Button variant="text" size="small">
  Text Only
</Button>
```

```tsx
<Button variant="text" size="small" disabled>
  Text Only
</Button>
```
