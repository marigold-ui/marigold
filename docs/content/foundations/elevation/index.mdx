---
title: Elevation
description: How surfaces and shadows work together to create visual depth and hierarchy
badge: updated
---

Screens are flat, but interfaces need depth. Without visual cues, every element sits on the same plane and users have to rely on position alone to understand what belongs together, what is interactive, and what is temporary. Elevation solves this by creating the illusion of layers: a card feels like it sits above the page, a dialog floats above everything else, and an input has a clear boundary against the surface it lives on.

This matters because it directly guides attention. A raised card groups related content and separates it from the surrounding page. A floating menu signals that it is temporary and will go away. A subtle border on an input tells users where they can type. Without these depth cues, interfaces feel flat and ambiguous.

Two systems work together to produce this effect: **background layers** set the color of each level, and **shadows** add visual lift to communicate how high something sits. For interactive elements like inputs and cards, a background is paired with a subtle border gradient and a matching shadow to complete the look.

<Callout title="Do I need to apply elevation manually?">
  Marigold components like `<Card>`, `<Dialog>`, and `<Menu>` already apply the correct background, border, and shadow tokens internally. You only need the utility classes described on this page (`ui-surface`, `shadow-elevation-*`, `bg-background`, etc.) when building custom elements that aren't covered by an existing component.
</Callout>

## Background layers

Before shadows come into play, background colors establish the base layers of the interface. Three tokens define where content sits:

<TokenTable
  tokens={[
    {
      name: 'background',
      value: 'charcoal-100',
      useFor: 'Page base layer, the gray canvas behind everything',
    },
    {
      name: 'surface',
      value: 'white',
      useFor: 'Content surface for cards, inputs, and panels',
    },
    {
      name: 'muted',
      value: 'charcoal-50',
      useFor: 'Subtle tint to set areas apart within a surface',
    },
  ]}
/>

`bg-background` and `bg-surface` work as a pair: the page background provides contrast so that white surfaces stand out. `bg-muted` adds a gentle tint within a surface to distinguish secondary content without adding another elevation level.

<ComponentDemo name="elevation-layers" mode="preview" />

For interactive elements like inputs, cards, and dialogs, the `ui-surface`
utility combines several properties into one class: a white background
(`bg-surface`), a subtle top-to-bottom border gradient that darkens slightly
toward the bottom edge, rounded corners, and smooth transitions for background,
border, and shadow changes. This gradient border gives elements a refined,
three-dimensional feel that a flat `1px solid` border cannot achieve. Pair
`ui-surface` with a `shadow-elevation-*` token to complete the elevation
styling:

```html
<div class="ui-surface shadow-elevation-raised">...</div>
```

## Shadow tiers

Elevation is split into three tiers, each serving a distinct role in the visual hierarchy. The shadow intensity increases with each tier: **border** provides a subtle edge for interactive controls, **raised** lifts containers off the page, and **overlay** creates a strong floating effect for temporary UI. Together they form a clear stacking order where every element sits at the right depth.

<ComponentDemo name="elevation-swatches" mode="preview" />

| Token                      | Use for                    | Components                                  |
| -------------------------- | -------------------------- | ------------------------------------------- |
| `shadow-elevation-border`  | Small interactive controls | Input, Checkbox, Radio, Switch, Tag, Button |
| `shadow-elevation-raised`  | Container surfaces         | Card, Accordion                             |
| `shadow-elevation-overlay` | Overlay elements           | Dialog, Drawer, Menu, Popover, Toast        |

This diagram shows how the three shadow tiers layer in a typical form layout. The page background (`bg-background`) is the base. A card (`shadow-elevation-raised`) groups form controls (`shadow-elevation-border`), while a floating panel (`shadow-elevation-overlay`) sits above everything.

![Annotated UI showing elevation tiers on a form layout](/elevation/elevation-in-context.svg)

### Border

The subtlest shadow tier. Border elevation gives small interactive controls a slight lift, defining their edges against the surface they sit on. These elements combine `ui-surface` with `shadow-elevation-border`.

Because border elevation produces only a subtle shadow, these controls need a raised surface behind them to be clearly visible. Placing an input directly on the page background (`bg-background`) makes its edges nearly indistinguishable from the surrounding area. A Card or similar raised container provides the contrast that border-elevated controls rely on.

<ComponentDemo name="elevation-border" />

```html
<div class="ui-surface shadow-elevation-border">...</div>
```

<GuidelineTiles>
  <Do>
    <DoDescription>
      Place form controls like inputs, checkboxes, and selects inside a raised
      surface such as a Card.
    </DoDescription>
  </Do>
  <Dont>
    <DontDescription>
      Don't place form controls directly on the page background without a raised
      container.
    </DontDescription>
  </Dont>
</GuidelineTiles>

### Raised

One step higher than border. Raised elevation creates visual lift for container surfaces that group related content. Cards and accordions use this tier to stand out from the page background while staying below floating elements. The flat page background behind them is what makes the shadow visible. Without that contrast, the lift disappears.

<ComponentDemo name="elevation-raised" />

```html
<div class="ui-surface shadow-elevation-raised">...</div>
```

<GuidelineTiles>
  <Do>
    <DoDescription>
      Place raised surfaces on the page background (`bg-background`) so the
      shadow creates clear visual lift.
    </DoDescription>
  </Do>
  <Dont>
    <DontDescription>
      Don't stack raised surfaces inside each other. The similar shadow levels
      make them visually indistinguishable.
    </DontDescription>
  </Dont>
</GuidelineTiles>

### Overlay

The strongest shadow tier, reserved for elements that temporarily float above the page. Dialogs, drawers, menus, popovers, and toasts use overlay elevation. Unlike the other tiers, overlays can stack on top of one another: a confirmation dialog over a menu, or a tooltip inside a drawer. They are always dismissible and often paired with a backdrop that dims the content underneath.

<ComponentDemo name="elevation-overlay" />

```html
<div class="ui-surface shadow-elevation-overlay">...</div>
```

<GuidelineTiles>
  <Do>
    <DoDescription>
      Use overlay elevation for temporary, dismissible elements that the user
      can close or navigate away from.
    </DoDescription>
  </Do>
  <Dont>
    <DontDescription>
      Don't use overlay elevation for persistent page content. If it's always
      visible, use raised elevation instead.
    </DontDescription>
  </Dont>
</GuidelineTiles>
