## Which setup?

Choose based on your goal:

- **[Tailwind CSS](#tailwind-css)** _(recommended for production)_ — full design system with tree-shaking, the same token scale (spacing, colors, breakpoints) in your own code, and integrated dev tooling (IntelliSense, JIT).
- **[Pre-built CSS](#pre-built-css)** _(prototyping only)_ — single import, no build configuration. Use for quick prototypes, AI-generated experiments, or environments where you can't run a build.

## Tailwind CSS

Marigold requires **Tailwind CSS v4**. If you're starting a new project, follow the official guides first:

- [Tailwind + Vite](https://tailwindcss.com/docs/installation/using-vite)
- [Tailwind + Next.js](https://tailwindcss.com/docs/installation/using-postcss) (the Next.js installer can do this for you)

Once Tailwind is set up, follow the steps for your framework:

{/* prettier-ignore-start */}

<Tabs items={['Vite', 'Next.js']}>

<Tab value="Vite">

<Steps>

<Step>

**Configure your stylesheet**

In `src/index.css`:

```css
@import 'tailwindcss';
@import '@marigold/theme-rui/preflight.css';
@import '@marigold/theme-rui/theme.css';

@source '../node_modules/@marigold';
```

The `@source` path is relative to the CSS file's location — adjust it if your entry CSS lives elsewhere.

**Why `preflight.css`?** It applies peer-dependency fixes on `<html>` and `<body>` that cannot ride along in `theme.css` — the theme bundle re-scopes every selector to `[data-theme="rui"]` so it can be adopted on a subtree, which would strip rules that must reach the real root elements.

</Step>

<Step>

**Wrap your app with `MarigoldProvider`**

This makes the theme available to every Marigold component.

```tsx
import { MarigoldProvider } from '@marigold/components';
import theme from '@marigold/theme-rui';

export const App = () => (
  <MarigoldProvider theme={theme}>{/* Your App */}</MarigoldProvider>
);
```

</Step>

<Step>

**Add the Inter font**

Install [Fontsource](https://fontsource.org/docs/getting-started/install):

```bash
npm install @fontsource-variable/inter
```

Then import it at your app's entry point:

```tsx
import '@fontsource-variable/inter';
```

</Step>

</Steps>

</Tab>

<Tab value="Next.js">

<Steps>

<Step>

**Configure your stylesheet**

In `src/app/globals.css`:

```css
@import 'tailwindcss';
@import '@marigold/theme-rui/preflight.css';
@import '@marigold/theme-rui/theme.css';

@source '../../node_modules/@marigold';
```

The `@source` path is relative to the CSS file's location — adjust it if your entry CSS lives elsewhere.

**Why `preflight.css`?** It applies peer-dependency fixes on `<html>` and `<body>` that cannot ride along in `theme.css` — the theme bundle re-scopes every selector to `[data-theme="rui"]` so it can be adopted on a subtree, which would strip rules that must reach the real root elements.

</Step>

<Step>

**Wrap your app with `MarigoldProvider`**

This makes the theme available to every Marigold component. Create a `providers.tsx` — the `'use client'` directive is required because `MarigoldProvider` uses React Context:

```tsx
'use client';
import { MarigoldProvider } from '@marigold/components';
import theme from '@marigold/theme-rui';

export function Providers({ children }: { children: React.ReactNode }) {
  return <MarigoldProvider theme={theme}>{children}</MarigoldProvider>;
}
```

Then wrap your app with `<Providers>` in `layout.tsx`.

</Step>

<Step>

**Add the Inter font**

Use [`next/font`](https://nextjs.org/docs/pages/building-your-application/optimizing/fonts) in `layout.tsx`:

```tsx
import { Inter } from 'next/font/google';

const inter = Inter({ subsets: ['latin'] });

// ...

<html className={inter.className}>
```

</Step>

## Pre-built CSS

Import the pre-built CSS that ships with the theme:

```tsx
import { MarigoldProvider } from '@marigold/components';
import theme from '@marigold/theme-rui';
import '@marigold/theme-rui/preflight.css';
import '@marigold/theme-rui/styles.css';

export const App = () => (
  <MarigoldProvider theme={theme}>{/* Your App */}</MarigoldProvider>
);
```

Add `data-theme="rui"` to your `<html>` element:

```html
<html data-theme="rui">
  ...
</html>
```

The pre-built bundle re-scopes every selector to `[data-theme="rui"]` so Marigold can be adopted on a subtree of a non-Marigold app. Without the attribute, component styles won't apply. The separate `preflight.css` import handles document-level fixes that can't be scoped — see [step 1 above](#1-configure-your-stylesheet) for the rationale.

## Opinionated defaults (opt-in)

Marigold ships two additional stylesheets that work with either setup above:

```css
/* Brand typography on <body> */
@import '@marigold/theme-rui/global.css';

/* Themed scrollbar on <html> — matches the `ui-scrollbar` utility */
@import '@marigold/theme-rui/page-scrollbar.css';
```

Neither is required. Skip them if you prefer your own typography or scrollbar styling.
