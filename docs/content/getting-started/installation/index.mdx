Marigold requires two packages: a core component library and a theme. Components ship unstyled — the theme provides all visual styling.

<Callout title="Want to try it first?">
  Use our [interactive playground](https://play.marigold-ui.io/) to explore
  Marigold without any setup.
</Callout>

## Install packages

Install the component library, the system package, and the RUI theme:

<Tabs items={['npm', 'pnpm']}>
  <Tab value="npm">
    ```bash npm install @marigold/components @marigold/system
    @marigold/theme-rui ```
  </Tab>
  <Tab value="pnpm">
    ```bash pnpm add @marigold/components @marigold/system @marigold/theme-rui
    ```
  </Tab>
</Tabs>

Marigold also provides an additional package you can install as needed:

- `@marigold/icons` — A set of ready-to-use icon components

If you use TypeScript, set [`moduleResolution`](https://www.typescriptlang.org/tsconfig#moduleResolution) to `"nodenext"` in your `tsconfig.json` — the theme packages use ESM exports.

## Set up the provider

Wrap your app with `MarigoldProvider` and pass it the theme. This is required for all components to render correctly.

How you load the CSS depends on whether you use Tailwind CSS or not. Pick one of the two approaches below.

### Quick start (pre-built CSS)

Import the pre-built CSS file that ships with the theme. This is the fastest way to get started but does not allow customization beyond the provided design tokens.

```jsx
import { MarigoldProvider } from '@marigold/components';
import theme from '@marigold/theme-rui';
import '@marigold/theme-rui/styles.css';

export const App = () => (
  <MarigoldProvider theme={theme}>{/* Your App */}</MarigoldProvider>
);
```

You also need to add `data-theme="rui"` to your `<html>` element so the theme styles are applied correctly:

```html
<html data-theme="rui">
  ...
</html>
```

### With Tailwind CSS (recommended)

Using Tailwind CSS alongside Marigold lets you customize and extend the theme and use Tailwind utilities throughout your app.

First, install Tailwind CSS. Follow the official [installation guide](https://tailwindcss.com/docs/installation) for your setup. Then create a CSS file (e.g. `styles.css`) with the following content:

```css
@import 'tailwindcss';
@import '@marigold/theme-rui/theme.css';

/* You need to add the path to the marigold packages */
@source '../node_modules/@marigold';
```

<Callout title="Next.js">
  In Next.js projects the CSS file typically lives in `app/`, which is one level
  deeper. Use `../../node_modules/@marigold` instead.
</Callout>

Make sure to import this CSS file in your app's entry point (e.g. your root layout or main file).

**Vite**: Add the `@tailwindcss/vite` plugin to your `vite.config.ts`. See the [Tailwind Vite guide](https://tailwindcss.com/docs/installation/using-vite).

**Next.js / PostCSS**: Install `@tailwindcss/postcss` and create a `postcss.config.mjs`:

```js
export default {
  plugins: {
    '@tailwindcss/postcss': {},
  },
};
```

Then wrap your app with the provider — no separate CSS import needed since Tailwind generates it for you:

{/* prettier-ignore-start */}
```jsx
import { MarigoldProvider } from '@marigold/components';
import theme from '@marigold/theme-rui';

export const App = () => (
  <MarigoldProvider theme={theme}>{/* Your App */}</MarigoldProvider>
);
```
{/* prettier-ignore-end */}

## Adding fonts

The RUI theme uses the [Inter](https://rsms.me/inter/) font family. You have two options:

**Using [Font Source](https://fontsource.org/docs/getting-started/install):**

```bash
npm install @fontsource-variable/inter
```

Then import it in your app's entry point:

```js
import '@fontsource-variable/inter';
```

**Using Next.js:** Take advantage of [`next/font`](https://nextjs.org/docs/pages/building-your-application/optimizing/fonts) for automatic font optimization.
