## Which setup?

Choose based on how Marigold fits into your app:

- **[Tailwind CSS](#tailwind-css)** _(recommended for new apps)_ — Marigold owns the page. Full design system with tree-shaking, the same token scale (spacing, colors, breakpoints) in your own code, integrated dev tooling (IntelliSense, JIT), and brand typography applied automatically.
- **[Pre-built CSS](#pre-built-css)** — Marigold is adopted on a subtree of an existing app, or you want a single CSS import with no build configuration (prototyping, AI-generated experiments, environments without a build step). Styles are scoped to `[data-theme="rui"]` so they never touch the host app's chrome.

## Tailwind CSS

Marigold requires **Tailwind CSS v4**. If you're starting a new project, follow the official guides first:

- [Tailwind + Vite](https://tailwindcss.com/docs/installation/using-vite)
- [Tailwind + Next.js](https://tailwindcss.com/docs/installation/using-postcss) (the Next.js installer can do this for you)

Once Tailwind is set up, follow the steps for your framework:

<Step>

**Configure your stylesheet**

In `src/index.css`:

```css
@import 'tailwindcss';
@import '@marigold/theme-rui/theme.css';

The `@source` path is relative to the CSS file's location — adjust it if your entry CSS lives elsewhere.

**Wrap your app with `MarigoldProvider`**

'use client';
import { MarigoldProvider } from '@marigold/components';
import theme from '@marigold/theme-rui';

export function Providers({ children }: { children: React.ReactNode }) {
  return <MarigoldProvider theme={theme}>{children}</MarigoldProvider>;
}
```

Then wrap your app with `<Providers>` in `layout.tsx`.

## Pre-built CSS

One import, no build configuration. The bundle carries component styles scoped to `[data-theme="rui"]` and the peer-dependency fixes on `<html>`/`<body>` un-scoped so they reach the real document root:

```tsx
import { MarigoldProvider } from '@marigold/components';
import theme from '@marigold/theme-rui';
import '@marigold/theme-rui/styles.css';

export const App = () => (
  <MarigoldProvider theme={theme}>{/* Your App */}</MarigoldProvider>
);
```

Mark the scoping element with `data-theme="rui"` — usually `<html>`, but any ancestor of the Marigold subtree works:

```html
<html data-theme="rui">
  ...
</html>
```

Without the attribute, component styles won't apply.

**Brand typography is opt-in here.** The Tailwind setup applies it by default; the pre-built bundle omits it on purpose — touching `<body>` typography from an embedded widget would be hostile to the host app. Add one import if you want it:

```tsx
import '@marigold/theme-rui/global.css';
```

## Advanced: composing your own bundle

The two import paths above are composition files. Pull individual pieces when you need a subset (for example, tokens in a non-Marigold context, or your own arrangement of the base rules):

| Path                                | Contents                                                                                                    |
| ----------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| `@marigold/theme-rui/theme.css`     | Composed Tailwind entry — tokens + utilities + variants + preflight + typography + RAC plugin               |
| `@marigold/theme-rui/styles.css`    | Pre-built scoped bundle — compiled Tailwind output, prefixed with `[data-theme="rui"]`, preflight un-scoped |
| `@marigold/theme-rui/tokens.css`    | Design tokens only (`@theme static` block)                                                                  |
| `@marigold/theme-rui/utilities.css` | `ui-*` utility classes                                                                                      |
| `@marigold/theme-rui/variants.css`  | Custom Tailwind variants                                                                                    |
| `@marigold/theme-rui/preflight.css` | Peer-dependency fixes on `<html>`/`<body>`                                                                  |
| `@marigold/theme-rui/global.css`    | Brand typography on `<body>`                                                                                |
